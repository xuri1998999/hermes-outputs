# PDF to Markdown 转换工具

## 功能概述

这是一个强大的 PDF 批量转换工具，使用 `pymupdf4llm` 库将 PDF 文档智能转换为结构化的 Markdown 文件。特别适合在 Obsidian 等知识管理系统中使用。

### 核心特性

- 📄 **批量转换**：一次处理目录下所有 PDF 文件
- 🌐 **完全支持中文**：包括中文标题、内容、元数据
- ⚡ **智能跳过**：自动识别已转换的文件，避免重复处理
- 📝 **元数据管理**：自动生成 YAML 前置元数据（source 字段）
- 📊 **进度提示**：彩色输出，清晰展示转换过程和统计结果
- 🛡️ **容错处理**：异常捕获，友好的错误提示

## 安装依赖

```bash
pip install pymupdf4llm --break-system-packages
```

或使用 pip 的现代方式：

```bash
pip install pymupdf4llm
```

## 使用方法

### 基本使用

```bash
python pdf2md.py
```

脚本会自动：
1. 扫描 `D:\ObsidianVaults\MyVault\pdfs\` 目录下所有 PDF 文件
2. 逐个转换为 Markdown 格式
3. 将输出文件保存在同一目录，同名但扩展名为 `.md`

### 自定义配置

编辑 `pdf2md.py` 中的 `main()` 函数：

```python
pdf_dir = r"D:\path\to\your\pdf\folder"  # 修改此路径
```

## 输出格式

转换后的 Markdown 文件格式如下：

```markdown
---
source: 原文件名.pdf
---

# 转换后的内容

正文内容...
```

## 故障排除

### 问题：找不到 PDF 目录
**解决**：检查 `pdf_dir` 路径是否正确，确保目录存在

### 问题：转换失败 ❌
**可能原因**：
- PDF 文件损坏或格式特殊
- 权限问题
- 内存不足

**解决**：查看错误提示信息，检查特定 PDF 文件

### 问题：中文字符显示乱码
**解决**：确保：
- Python 文件编码为 UTF-8
- 文件系统支持中文
- 输出文件使用 UTF-8 编码（代码已设置）

## 性能考虑

- 单个 PDF 转换时间取决于文件大小和页数
- 建议在本地运行以获得最佳性能
- 对于大写 PDF 仓库，可在后台运行
